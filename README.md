# CS 360 Portfolio Artifact – Smart Event Tracker

## Project Summary

The mobile application I developed for this course was the Smart Event Tracker app. The goal of the app was to help users organize, manage, and track personal events simply and practically. The app was designed to address user needs related to scheduling, organization, and reminders by allowing users to log in, create events, view event details, edit events, and receive notifications. A major goal of the project was not only to build a functional app, but also to apply user-centered design principles so the app would be easy to use and intuitive.

To support user needs, several screens and features were necessary, including a login screen, event list screen, add event screen, event detail/edit screen, and notification permissions screen. My UI designs focused on keeping navigation simple, organizing information clearly, and making features easy to access. I tried to keep the layouts clean and functional so users could move through the app without confusion. I believe the designs were successful because they balanced usability with functionality and kept the user experience at the center of the design.

## Development Process

When coding the application, I approached development step by step by building and testing one feature at a time. I focused on connecting layouts, Java code, user input handling, and database functionality so all parts of the application worked together. Some strategies I used included breaking problems into smaller tasks, debugging incrementally, and improving features as I tested them. These are techniques I can continue applying in future software development projects because they support organized development and make troubleshooting easier.

Testing was an important part of the process. I used testing throughout development by running the application, checking functionality in the Android Emulator, reviewing user interactions, and debugging errors as they appeared. This process was important because it helped reveal issues in logic, navigation flow, and feature behavior that needed improvement. Testing helped ensure the app functioned as intended and reinforced how important iteration is in software development.

## Challenges and Innovation

One challenge during the full design and development process was bringing the user interface design, event management features, database functionality, and notifications together into one working application. I had to think creatively about connecting design ideas with practical implementation and refining features as I tested them. That process reinforced how much iteration and problem-solving are involved in mobile development.

Innovation also came through refining the app design, improving functionality, and finding ways to make the application more complete and user-focused.

## Demonstrated Knowledge and Skills

One area where I was particularly successful was demonstrating user-centered design, together with coding best practices and functional development. This project allowed me to apply planning, UI design, coding, debugging, and testing in one complete application. It reflects both technical growth and a stronger understanding of how mobile applications are designed and developed from concept to launch.
